# Contributing to Effort Tracker

Thank you for your interest in contributing to Effort Tracker! This document provides guidelines for contributing to this issue tracker repository.

## How to Contribute

### Reporting Bugs

1. **Search existing issues** to avoid duplicates
2. **Use the bug report template** when creating a new issue
3. **Provide detailed information:**
   - Steps to reproduce
   - Expected vs actual behavior
   - Environment details (browser, Azure DevOps version, extension version)
   - Screenshots if applicable

### Requesting Features

1. **Check existing feature requests** to see if it's already been suggested
2. **Use the feature request template**
3. **Describe the use case** - explain why this feature would be valuable
4. **Be specific** about what you'd like to see

### Improving Documentation

Found a typo or unclear documentation? Feel free to:
- Open an issue describing what needs improvement
- Suggest specific changes

### Providing Context

The more context you provide, the faster we can help:
- Include version numbers
- Add screenshots or screen recordings
- Describe your Azure DevOps setup (Services vs Server)
- Mention any custom configurations

## Response Times

- **Bug reports:** We aim to triage with our best efforts
- **Feature requests:** Reviewed during sprint planning
- **Security issues:** Addressed with highest priority (see [SECURITY.md](SECURITY.md))

## Code of Conduct

Please read and follow our [Code of Conduct](CODE_OF_CONDUCT.md).

## Questions?

For general questions, please use [GitHub Discussions](https://github.com/Sinaware/EffortTracker/discussions) instead of creating an issue.

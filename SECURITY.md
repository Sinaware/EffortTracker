# Security Policy

## Reporting a Vulnerability

We take the security of Effort Tracker seriously. If you discover a security vulnerability, please report it responsibly.

### How to Report

**Please do NOT report security vulnerabilities through public GitHub issues.**

Instead, please report them via email to: sinagulsen@gmail.com

Include the following information in your report:

- Type of vulnerability
- Full paths of source file(s) related to the vulnerability (if known)
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the vulnerability

### What to Expect

- **Acknowledgment:** We will acknowledge receipt of your vulnerability with email.
- **Updates:** We will keep you informed of our progress toward a fix.
- **Resolution:** Once we have determined the impact and severity, we will work on a fix and coordinate disclosure with you.

### Scope

This security policy applies to:

- Effort Tracker Azure DevOps Extension
- Associated backend services

### Out of Scope

- Third-party dependencies (report to the respective maintainers)
- Azure DevOps platform vulnerabilities (report to Microsoft)

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| Latest  | ✅ Yes             |
| < Latest| ❌ No              |

We recommend always using the latest version of the extension.

## Security Best Practices for Users

1. **Keep the extension updated** to the latest version
2. **Use appropriate permissions** when configuring the extension
3. **Review activity logs** regularly for unusual activity
4. **Report suspicious behavior** immediately

Thank you for helping keep Effort Tracker and our users safe!
